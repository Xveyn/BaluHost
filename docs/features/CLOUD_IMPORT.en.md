# Cloud Import Setup

## Prerequisites

Before you can import files from Google Drive or OneDrive, you need to set up an OAuth connection with the cloud provider. You will need:

1. A **Google Cloud** or **Microsoft Azure** account (free)
2. OAuth credentials (Client ID + Client Secret) from the provider
3. A valid **redirect URI** configured in the provider's console

---

## Setting up Google Drive

### Step 1: Create a Google Cloud Project

1. Go to the [Google Cloud Console](https://console.cloud.google.com)
2. Create a new project (or use an existing one)
3. Enable the **Google Drive API**:
   - Navigate to APIs & Services > Library
   - Search for "Google Drive API" and enable it

### Step 2: Create OAuth Credentials

1. Go to **APIs & Services > Credentials**
2. Click **"+ Create Credentials" > "OAuth client ID"**
3. Select **"Web application"** as the application type
4. Enter a name (e.g., "BaluHost")
5. Under **"Authorized redirect URIs"**, add:
   ```
   https://<your-domain>/api/cloud/oauth/callback
   ```

> **Important:** Google does **not** accept `.local` domains or IP addresses as redirect URIs. You need a domain with a public TLD (e.g., `.org`, `.com`, `.net`).
>
> **Recommendation:** Use a free DNS service like [DuckDNS](https://www.duckdns.org):
> 1. Create a subdomain (e.g., `mynas.duckdns.org`)
> 2. Point it to your **local NAS IP** (e.g., `192.168.178.20`)
> 3. Use `https://mynas.duckdns.org/api/cloud/oauth/callback` as the redirect URI
> 4. Add an entry to your PC's hosts file (`C:\Windows\System32\drivers\etc\hosts`):
>    ```
>    192.168.178.20 mynas.duckdns.org
>    ```
> 5. Set the environment variable `PUBLIC_URL=https://mynas.duckdns.org` on the NAS

6. Click **"Create"**
7. Copy the **Client ID** and **Client Secret**

### Step 3: Configure the OAuth Consent Screen

1. Go to **APIs & Services > OAuth consent screen**
2. Select **"External"** as the user type
3. Fill in the required details (app name, support email)
4. Under **"Test users"**, add your own Google email address (while the app is unpublished, only test users can sign in)

### Step 4: Enter Credentials in BaluHost

1. Go to **Settings > Integrations** in BaluHost
2. Click **"Configure"** on Google Drive
3. Enter the Client ID and Client Secret
4. Click **"Save"**

### Step 5: Connect

1. Go to **Cloud Import** (sidebar)
2. Click **"Connect"** and select Google Drive
3. You will be redirected to Google — sign in and grant permission
4. After successful authorization, you will be redirected back to BaluHost

## Setting up OneDrive

### Step 1: Register an Azure App

1. Go to the [Azure Portal](https://portal.azure.com) > **Azure Active Directory > App registrations**
2. Click **"New registration"**
3. Enter a name (e.g., "BaluHost")
4. Under **Supported account types**, select **"Accounts in any organizational directory and personal Microsoft accounts"**
5. Under **Redirect URI**, select "Web" and enter:
   ```
   https://<your-domain>/api/cloud/oauth/callback
   ```
6. Click **"Register"**

### Step 2: Create a Client Secret

1. Go to **Certificates & secrets**
2. Click **"New client secret"**
3. Enter a description and choose an expiration period
4. Copy the **Value** (shown only once!)

### Step 3: Enter Credentials in BaluHost

1. Go to **Settings > Integrations** in BaluHost
2. Click **"Configure"** on OneDrive
3. Enter the **Application (client) ID** and the **Client Secret**
4. Click **"Save"**

### Step 4: Connect

Same process as Google Drive — connect via Cloud Import.

---

## Importing Files

After a successful connection:

1. Go to **Cloud Import**
2. Select the desired connection
3. Browse your cloud files
4. Select files/folders to import
5. Choose the destination folder on the NAS
6. Start the import — progress is shown in real time

## Common Issues

### "redirect_uri_mismatch" Error

The redirect URI in the cloud console does not exactly match the URI generated by BaluHost. Check:

- Does the protocol match? (`http` vs `https`)
- Does the domain match exactly? (including port, if any)
- Is the `PUBLIC_URL` environment variable set correctly on the NAS?
- Did you restart the backend after changing it?

### "Server not found" After Google Login

Your browser cannot resolve the domain. Check:

- Is the domain (e.g., DuckDNS) pointing to the correct local IP?
- Is there an entry in your PC's hosts file?
- Run `ipconfig /flushdns` (Windows) or `sudo dscacheutil -flushcache` (macOS)

### Google Shows "Access Blocked"

- Check if your email is added as a **Test User** in the OAuth Consent Screen
- Or publish the app (requires Google verification)

### Cannot Delete Connection

If you get a 500 error when deleting, restart the backend and try again.
