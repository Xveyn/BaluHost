Grösseneinordnung

Kontext

Wie gross ist die Codebase wirklich?

Mit ~152.000 Zeilen Code ist BaluHost ein grosses Einzelentwickler-Projekt. Zum Vergleich: Typische Open-Source-Projekte wie Gitea (Go-basierter Git-Server) haben ~200K LOC, während einfachere Self-Hosted-Tools wie Filebrowser bei ~30K LOC liegen.

Die Komplexität entsteht durch die Breite der Features: BaluHost vereint Dateiverwaltung, RAID-Management, VPN-Konfiguration, Power-Management, Fan-Steuerung, Sleep Mode, Monitoring, Backup, Sync, Notifications, File Versioning, SSD File Cache, Plugin-System und Mobile-App-Anbindung in einer einzigen Plattform — ein Umfang, den man typischerweise bei Teams von 3–5 Entwicklern erwarten würde.

Fazit

BaluHost hat seit v2 einen qualitativen Sprung gemacht — und erreicht erstmals die Höchstnote A+. Die ~152.000 Zeilen Code (+13K seit v2) decken ein Feature-Set ab, das typischerweise von kleinen Teams entwickelt wird.

Der wichtigste Fortschritt seit v2 ist die systematische Backend-Refactoring-Welle: 7 grosse Services (benchmark, smart, raid, update, power/manager, files, scheduler) wurden in saubere Sub-Packages aufgeteilt, die alle dem gleichen Pattern folgen. Gleichzeitig wurden bedeutende neue Features integriert — ein vollständiges Notification-System, File Versioning (VCL), SSD File Cache, Desktop Pairing und API Key Management — ohne die Code-Qualität zu beeinträchtigen.

Die any-Typen wurden um fast die Hälfte reduziert (81→42), das Audit-Logging mehr als verdoppelt (109→253), Rate-Limited Endpoints um 25% erhöht (24→30). Das Multi-Worker Deployment mit Primary Worker Guard und DB-Fallbacks zeigt produktionsreife Infrastruktur.

Gesamtbewertung: A+ — Hervorragende, produktionsreife Codebase mit systematischer Architekturverbesserung, starkem Security-Fokus und messbaren Qualitätssteigerungen über drei Report-Iterationen.
